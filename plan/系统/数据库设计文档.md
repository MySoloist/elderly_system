# 老年人社区餐饮配送系统 - 数据库设计文档

## 1. 概述

本文档详细描述老年人社区餐饮配送系统的数据库设计方案，基于PostgreSQL + PostGIS技术栈，支持地理空间功能和多端共享架构。

## 2. 数据库架构

### 2.1 数据库版本
- **PostgreSQL**: 15.0+
- **PostGIS**: 3.3.0+

### 2.2 枚举类型定义

```sql

    id SERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    user_type usertype NOT NULL, -- elderly, member, deliverer, admin
    status userstatus DEFAULT 'ACTIVE',
    email VARCHAR(100),
    last_login TIMESTAMPTZ,
    openid VARCHAR(100),
    unionid VARCHAR(100),
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW(),
    CONSTRAINT check_user_type CHECK (user_type IN ('elderly', 'member', 'deliverer', 'admin'))
);
```

## 3. 触发器和函数

### 3.1 自动更新时间戳函数
```sql
CREATE OR REPLACE FUNCTION update_timestamp()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = NOW();
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;
```

### 3.2 为用户表创建更新时间戳触发器
```sql
CREATE TRIGGER trigger_users_updated_at
BEFORE UPDATE ON users
FOR EACH ROW
EXECUTE FUNCTION update_timestamp();
```

### 3.3 为老人资料表创建更新时间戳触发器
```sql
CREATE TRIGGER trigger_elderly_profiles_updated_at
BEFORE UPDATE ON elderly_profiles
FOR EACH ROW
EXECUTE FUNCTION update_timestamp();
```

### 3.4 为配送员资料表创建更新时间戳触发器
```sql
CREATE TRIGGER trigger_deliverer_profiles_updated_at
BEFORE UPDATE ON deliverer_profiles
FOR EACH ROW
EXECUTE FUNCTION update_timestamp();
```

## 4. 数据完整性约束

### 4.1 外键约束

所有表都设置了适当的外键约束，确保数据完整性：
- 用户表与各用户类型表的一对一关系
- 订单表与用户表的多对一关系
- 订单项与订单表、餐品表的多对一关系
- 配送表与订单表、用户表的多对一关系

### 4.2 检查约束

```sql
-- 用户类型检查
ALTER TABLE users ADD CONSTRAINT check_user_type 
CHECK (user_type IN ('elderly', 'member', 'deliverer', 'admin'));

-- 订单状态检查
ALTER TABLE orders ADD CONSTRAINT check_order_status 
CHECK (status IN ('pending_payment', 'pending_schedule', 'pending_accept', 'delivering', 'completed', 'cancelled'));

-- 配送员状态检查
ALTER TABLE deliverer_profiles ADD CONSTRAINT check_deliverer_status 
CHECK (status IN ('available', 'busy', 'offline', 'admin_online'));

-- 评分检查
ALTER TABLE reviews ADD CONSTRAINT check_rating 
CHECK (rating >= 1 AND rating <= 5);

-- 评价者类型检查
ALTER TABLE reviews ADD CONSTRAINT reviews_reviewer_type_check 
CHECK (reviewer_type IN ('elderly', 'family'));
```

### 4.3 唯一约束

```sql
-- 用户名字段唯一
ALTER TABLE users ADD CONSTRAINT users_username_key UNIQUE (username);

-- 老人手机号唯一
ALTER TABLE elderly_profiles ADD CONSTRAINT elderly_profiles_phone_key UNIQUE (phone);

-- 家属手机号唯一
ALTER TABLE member_profiles ADD CONSTRAINT member_profiles_phone_key UNIQUE (phone);

-- 配送员手机号唯一
ALTER TABLE deliverer_profiles ADD CONSTRAINT deliverer_profiles_phone_key UNIQUE (phone);

-- 订单与评价一对一关系
ALTER TABLE reviews ADD CONSTRAINT reviews_order_id_key UNIQUE (order_id);

-- 老人与家属关系唯一
ALTER TABLE elder_member_relations ADD CONSTRAINT elder_member_relations_elder_id_member_id_key 
UNIQUE (elder_id, member_id);

-- 老人与餐品收藏唯一
ALTER TABLE favorites ADD CONSTRAINT favorites_elderly_id_meal_id_key 
UNIQUE (elderly_id, meal_id);

-- 员工排班唯一（同一天同一时间段）
ALTER TABLE staff_schedules ADD CONSTRAINT staff_schedules_staff_id_schedule_date_time_slot_key 
UNIQUE (staff_id, schedule_date, time_slot);
```
